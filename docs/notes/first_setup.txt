Here’s a solid starting point for an AI translation web app using Next.js (App Router) running on Bun ([bun.sh](http://bun.sh/)).

1. Scaffold Next.js with Bun
Bun has an official guide for running Next.js with Bun.[x]
bun create next-app@latest ai-translate
cd ai-translate
bun --bun run dev
Update package.json scripts so Next.js runs via Bun:[x]
{
"scripts": {
"dev": "bun --bun next dev",
"build": "bun --bun next build",
"start": "bun --bun next start"
}
}
2. Add an AI translation backend route (recommended)
Typical setup:
• UI page posts { text, from, to, tone? }
• Next.js route handler calls an LLM provider (OpenAI, etc.)
• Return translatedText
If you want a reference implementation, there’s an example repo that uses Next.js + Bun + Vercel AI SDK/OpenAI + Zod + Tailwind/shadcn and focuses specifically on translation features.[x]
3. UI features to include (minimal but useful)
• Source language + target language dropdowns
• Textarea input
• “Translate” button
• Output panel with copy button
• Basic validation (empty input, max chars)
4. Deploy
This kind of Next.js app is commonly deployed on Vercel (especially if using the Vercel AI SDK), but you can deploy anywhere Next.js runs.
